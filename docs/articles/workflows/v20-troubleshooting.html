Troubleshooting

This vignette collects common errors and fixes encountered when building or running DPmixGPD models.

1. “Number of columns in ‘x’ does not match training design matrix”

Cause: New data has different columns than the training X.

Fix: Use the same column names and order as the training matrix.

Code 
X_train <- fit$data$X
X_new <- X_train[1:5, , drop = FALSE]
X_new <- X_new[, colnames(X_train), drop = FALSE]

predict(fit, x = X_new, type = "mean")

2. “Cauchy kernels are never paired with GPD tails”

Cause: The Cauchy kernel is not allowed with GPD tails.

Fix: Use a different kernel or disable GPD.

Code 
bundle <- build_nimble_bundle(
  y = y,
  kernel = "cauchy",
  backend = "sb",
  GPD = FALSE
)

3. “design=‘observational’ requires non-empty X”

Cause: Observational causal modeling requires covariates for PS estimation.

Fix: Provide X, or switch to design = "rct".

Code 
cb <- build_causal_bundle(
  y = y,
  T = T,
  X = X,
  design = "observational",
  PS = "logit"
)

4. “GPD tail not moving” (threshold/shape stuck)

Cause: Very short chains or overly restrictive priors.

Fix: Increase iterations or relax tail priors.

Code 
bundle <- build_nimble_bundle(
  y = y,
  kernel = "gamma",
  backend = "crp",
  GPD = TRUE,
  components = 6,
  mcmc = mcmc
)

5. “future.globals.maxSize exceeded”

Cause: Parallel predictions export large functions/objects.

Fix: Set a higher limit or use ncores = 1.

Code 
options(future.globals.maxSize = 4 * 1024^3)

pred <- predict(fit, x = X_new, y = y_grid, type = "density", ncores = 2)

6. MCMC is slow

Cause: Large datasets, many components, or heavy kernels.

Fix: Use smaller components/J, shorter niter, or start with CRP.

Code 
bundle <- build_nimble_bundle(
  y = y,
  backend = "crp",
  kernel = "normal",
  components = 4,
  mcmc = mcmc
)

7. Prediction returns constant fitted values in unconditional models

Cause: Unconditional fitted values are model-based population summaries.

Fix: This is expected. Use conditional models if you need observation-specific fitted values.

Code 
fit_vals <- fitted(fit)
head(fit_vals)

8. PS not used in causal model

Cause: PS=FALSE or X missing.

Fix: Provide X and enable PS.

Code 
cb <- build_causal_bundle(
  y = y,
  T = T,
  X = X,
  PS = "logit"
)
fit <- run_mcmc_causal(cb)

9. Kernel mismatch when mixing arms

Cause: Using kernels with different supports across arms.

Fix: Ensure both arms use kernels on the same support (e.g., both positive-support or both real-line).

Code 
cb <- build_causal_bundle(
  y = y,
  T = T,
  X = X,
  kernel = c("gamma", "lognormal"),
  backend = c("sb", "crp")
)
