ex11. Causal (Same backend, CRP Backend) - Cauchy/Laplace

Website workflow note. This page reflects the current exported API and recommended wrapper-first usage. Last updated: 2026-02-19.

For the full package narrative, see the main package vignettes (basic, unconditional, conditional, and causal).

Causal Inference: Mixed Backends (CRP+SB)

This vignette fits two causal models using different backends and kernels:

  • Model A: CRP bulk-only (Cauchy)
  • Model B: SB with GPD tail (Laplace)

Each model demonstrates the full causal S3 workflow: bundle, fit, predict, plot, ATE, and QTE.

What you’ll learn

  • How backend choice (CRP vs SB) and tail choice (GPD = FALSE/TRUE) affect predictive uncertainty and downstream causal estimands.

  • How to keep comparisons interpretable by using the same post-fit S3 surface across models (summary(), predict(), plot()).

  • How to read both mean and quantile effect summaries and treat them as model-based functionals. ## When to use this template

  • You want an apples-to-apples comparison of causal models that differ by backend and/or tail augmentation.

  • You want a compact example of the full causal S3 workflow in one place. ## Next steps

  • Use the “same-backend” causal examples to isolate tail effects from backend effects.
