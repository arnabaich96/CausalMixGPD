\Abstract{
We present DPmixGPD, an R package for Bayesian nonparametric mixture modeling with explicit extreme-value tail behavior.
The package models the bulk distribution using a Dirichlet process mixture (DPM) with multiple kernel families and supports both Chinese restaurant process (CRP) and truncated stick-breaking (SB) backends.
For heavy-tailed outcomes, DPmixGPD optionally splices a generalized Pareto distribution (GPD) above a threshold, enabling coherent extrapolation for extreme quantiles and tail probabilities while retaining flexible bulk fit.
The workflow is designed around two main functions: build-nimble-bundle constructs a compiled NIMBLE model bundle, and run-mcmc-bundle-manual runs MCMC to produce a fitted object with print, summary, plot, fitted, and predict methods.
We demonstrate density estimation, conditional modeling with covariates, and a real-data application using the built-in airquality dataset, highlighting stable high-quantile inference and tail exceedance summaries.
}

\section{Introduction}

Real-world data are often shaped by two competing demands: (i) the \emph{bulk} of the distribution can exhibit complex structure such as skewness or multimodality, and (ii) the \emph{tail} may contain extreme outcomes that require principled extrapolation beyond observed values.
Bayesian nonparametric mixture models address (i) by allowing flexible densities without fixing the number of components in advance, typically through a Dirichlet process (DP) prior \citep{Ferguson1973, Neal2000}.
Extreme value theory addresses (ii) by characterizing exceedances above a high threshold via the generalized Pareto distribution (GPD) \citep{Pickands1975, DeHaanFerreira2006}.
The \pkg{DPmixGPD} package combines these ideas in a unified Bayesian framework.
The bulk distribution is modeled as a Dirichlet process mixture (DPM) with selectable kernels and backends.
Optionally, the upper tail is modeled using a GPD beyond a threshold, yielding a spliced distribution that preserves bulk flexibility while enforcing extreme-value tail behavior.
This design targets settings where bulk-only mixtures can be sensitive in extreme extrapolation (e.g., unstable very-high quantiles), and where purely parametric bulk+tail models are too rigid for the central distribution.
The package emphasizes a practical workflow:
\begin{enumerate}
  \item \code{build_nimble_bundle()} constructs a \pkg{NIMBLE}-based model bundle from data and a model specification.
  \item \code{run_mcmc_bundle_manual()} runs MCMC and returns a fitted object with S3 methods for summarization, diagnostics, and prediction.
\end{enumerate}
Users can compute posterior predictive densities, survival curves, and quantiles (including extreme quantiles) with coherent uncertainty quantification.

\section{Model formulation}

\subsection{Bulk model: Dirichlet process mixtures}

Let $y_1,\dots,y_n$ denote observations and $f(\cdot \mid \theta)$ a kernel density (e.g., lognormal, gamma, normal, Laplace, inverse Gaussian, Amoroso).
A Dirichlet process mixture model can be written hierarchically as
\begin{align}
y_i \mid \theta_i &\sim f(y_i \mid \theta_i), \\
\theta_i \mid G &\sim G, \\
G &\sim \mathrm{DP}(\alpha, G_0),
\end{align}
where $\alpha$ is the concentration parameter and $G_0$ is a base distribution for kernel parameters.
Marginally, this implies an (infinite) mixture representation
\[
p(y) = \sum_{j=1}^{\infty} w_j f(y \mid \Theta_j),
\]
with random weights $w_j$ and atoms $\Theta_j$ induced by the DP prior.

\subsection{Inference backends: CRP and stick-breaking}

\pkg{DPmixGPD} supports two standard representations:

\paragraph{CRP backend.}
A Chinese restaurant process (CRP) sampler integrates out $G$ and samples cluster allocations.
This representation is convenient when the number of occupied clusters is unknown and allows it to adapt automatically \citep{Neal2000}.

\paragraph{SB backend.}
A truncated stick-breaking (SB) representation approximates the infinite mixture with $J$ components, where weights follow a stick-breaking prior and $J$ is chosen sufficiently large.
The posterior typically concentrates on a smaller effective number of components.

\subsection{Tail model: generalized Pareto distribution}

To model extremes, \pkg{DPmixGPD} optionally splices a generalized Pareto distribution above a threshold $u$.
Let $F_{\text{bulk}}(y)$ denote the bulk CDF.
For $y>u$, define
\[
F(y) = F_{\text{bulk}}(u) + \{1-F_{\text{bulk}}(u)\} F_{\text{GPD}}(y;u,\sigma,\xi),
\]
where $F_{\text{GPD}}(\cdot)$ is the GPD CDF with scale $\sigma>0$ and shape $\xi$ \citep{Pickands1975, DeHaanFerreira2006}.
This formulation ensures a valid distribution with continuity at $u$ and interpretable tail behavior:
$\xi>0$ corresponds to heavy tails, $\xi=0$ to exponential-like tails, and $\xi<0$ to bounded tails.
The threshold $u$ can be fixed or inferred.
In many applications, the threshold is treated as unknown with a weakly informative prior centered near a high empirical quantile.
The resulting posterior uncertainty in $u$ is then propagated into tail quantiles and exceedance probabilities.

\subsection{Conditional modeling with covariates}

When covariates $x_i$ are available, \pkg{DPmixGPD} supports conditional modeling where selected kernel parameters (and optionally tail components) depend on $x_i$ through link functions.
In practice, \code{build_nimble_bundle()} accepts a design matrix \code{X} and constructs a conditional mixture model so that predictions can be computed for new covariate profiles.

\section{Implementation}

\pkg{DPmixGPD} is implemented in \proglang{R} and uses \pkg{NIMBLE} to express and compile probabilistic models.
The package contains:
\begin{itemize}
  \item A kernel registry with supported distributions for bulk modeling.
  \item Custom likelihood components for mixture densities and mixture+GPD splices.
  \item A model builder that generates \pkg{NIMBLE} code from a user specification, including priors and backends.
  \item An MCMC runner that executes compiled samplers and returns a fitted object with S3 methods.
\end{itemize}
The API is designed to be stable: the same user-facing functions cover unconditional, conditional, and tail-augmented models with minimal changes in arguments.

\subsection{Interpretation}

Across profiles, the bulk-only mixture typically captures central quantiles well but may be sensitive in high-quantile extrapolation depending on the kernel behavior in the far tail.
The bulk+GPD specification enforces an extreme-value tail form beyond the learned threshold, which can stabilize inference for the 0.95--0.99 quantiles and exceedance probabilities.
In applications where decisions depend on rare high outcomes (e.g., risk, reliability, safety), this tail regularization is often the main practical value of the model \citep{Pickands1975, DeHaanFerreira2006}.

\section{Discussion and future work}

\pkg{DPmixGPD} provides a unified Bayesian workflow for flexible bulk modeling and principled tail inference.
Compared to purely parametric bulk+tail approaches, the DPM bulk reduces risk of misspecifying central features such as multimodality or skewness.
Compared to bulk-only nonparametric mixtures, the explicit GPD tail can reduce instability in extreme quantiles.
Planned extensions include: (i) additional kernels and regression-link options, (ii) improved posterior predictive diagnostics for tail adequacy, (iii) computational refinements for larger datasets (e.g., faster sampling or approximate inference), and (iv) expanded causal and survival modeling modules where tail behavior is scientifically central.
